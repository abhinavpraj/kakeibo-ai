# User Growth & Continuous Improvement Plan

## Objective

KakeiboAI aims to become an AI-powered personal finance companion helping users build sustainable budgeting habits. Unlike traditional trackers, it focuses on behavioral reflection and understanding "why" spending occurs, empowering users to make intentional financial decisions and build long-term financial health.

## Current Stage

The project is currently at the following stage:
* **MVP completed**: Core tracking and budget features are fully operational.
* **Authentication implemented**: User sign-up and login capabilities are ready.
* **AI financial insights**: Monthly spending and saving advice is generated dynamically.
* **Local AI (Ollama)**: Local inference capability using `llama3` is supported.
* **Gemini BYOK support**: Cloud AI inference using Google Gemini API is supported via Bring Your Own Key (BYOK).
* **Feedback collection**: Integrated feedback aggregation using remote Supabase backend and Google Forms.
* **Multi-language support**: Localized interface available in English, Hindi (हिन्दी), Marathi (మరాఠీ), and Telugu (తెలుగు).

## Continuous Product Improvement

We are committed to an iterative development approach driven by user feedback. Based on feedback collected from early users, the following feature updates are prioritized for upcoming releases:
* **Better AI recommendations**: Refine contextual prompts for deeper financial behavior analysis.
* **Bank statement upload**: Support parsing CSV/Excel statements to automate transaction entry.
* **Expense categorization**: Implement automated categorization rules and predictive category assignments.
* **Financial reports**: Monthly summary sheets and visual trends exportable to PDF formats.
* **Better visualizations**: Add interactive plotly layouts for cashflow comparisons.
* **More languages**: Integrate additional Indian regional languages (e.g., Tamil, Kannada, Malayalam).
* **Mobile responsiveness**: Optimize Streamlit components to render seamlessly on lower-end mobile browsers.

## Feedback Collection Strategy

To drive continuous product improvements, user feedback is systematically collected through:
* **In-app feedback**: Localized call-to-action buttons directing users to feedback forms.
* **Star ratings**: Direct assessment of app usability.
* **Comments**: Qualitative suggestions on features and layouts.
* **Community reviews**: Feedback collected from open-source repositories and public deployment platforms.
* **Usage analytics**: Aggregated, anonymized statistics stored in Supabase to track active features.

## Success Metrics

We measure the growth and success of KakeiboAI using the following key performance indicators (KPIs):
* **Registered users**: Total number of user accounts created.
* **Daily active users (DAU)**: Number of unique users logging transactions daily.
* **Monthly active users (MAU)**: Unique monthly users reflecting on spending.
* **AI conversations**: Frequency of prompts sent to the Ollama and Gemini engines.
* **Feedback submissions**: Volume of completed survey forms.
* **Average user rating**: Goal of maintaining >4.2/5 stars.
* **User retention**: Percentage of users continuing to track budgets across consecutive months.
